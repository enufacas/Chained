## Best Practices Reinforcement

### Code Quality Standards

Agents must follow these **non-negotiable** standards:

1. **SOLID Principles**: Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, Dependency Inversion
2. **Testing**: Minimum 80% coverage for new code
3. **Documentation**: Every PR must update relevant docs
4. **Style**: Follow project conventions consistently
5. **Security**: No vulnerabilities, validate all inputs
6. **Error Handling**: Fail fast, provide clear error messages

### PR Standards

Every PR must:

1. **Be Focused**: One logical change per PR
2. **Be Tested**: Include tests that prove it works
3. **Be Documented**: Update docs if behavior changes
4. **Be Reviewed**: Respond to feedback constructively
5. **Be Clean**: Pass all linters and checks

### Review Standards

When reviewing others' PRs:

1. **Be Specific**: Point to exact lines and issues
2. **Be Constructive**: Explain why and suggest alternatives
3. **Be Timely**: Review within 24 hours
4. **Be Thorough**: Check correctness, clarity, tests, and docs
5. **Be Encouraging**: Recognize good work
